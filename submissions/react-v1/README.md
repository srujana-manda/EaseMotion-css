# ⚛️ React Submissions

This directory is for **React component** contributions to EaseMotion CSS.

## How to Submit

1. **Claim an issue** tagged `react` + `GSSoC-26` on [GitHub Issues](https://github.com/SAPTARSHI-coder/EaseMotion-css/issues?q=is:open+label:react)
2. **Fork** the repository and create a new branch
3. **Create a folder** here: `submissions/react/your-component-name/`
4. **Add your files** (see required structure below)
5. **Open a PR** — the React validator bot will check your submission automatically

## Required File Structure

```
submissions/react/your-component-name/
├── YourComponent.jsx   ← Your React component (required)
├── style.css           ← CSS styles (optional but encouraged)
└── README.md           ← Description, props table, usage example (required)
```

## Rules

| Rule | Details |
|------|---------|
| ✅ Only add new files | No editing or deleting existing files |
| ✅ React components only | `.jsx` or `.tsx` files |
| ✅ Use EaseMotion CSS | Must use `ease-*` class names from the framework |
| ✅ Original work | No copy-pasted boilerplate or AI mass-generation |
| ✅ One component per PR | Keep PRs focused |
| ❌ No plain HTML/CSS | Wrong track — use `submissions/examples/` instead |
| ❌ No external dependencies | Only React + EaseMotion CSS |

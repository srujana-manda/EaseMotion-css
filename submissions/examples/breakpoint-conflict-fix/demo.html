Responsive Mixins vs Custom Media Queries — Breakpoint Conflict Fix

Previously, the SCSS respond-to() mixin used one set of pixel breakpoints (e.g. tablet at 768px), while several component files wrote their own ad-hoc @media queries with slightly different values (e.g. tablet at 760px or 800px). Components built with the mixin and components with custom queries shifted layout at different viewport widths even though both were labeled "tablet," producing visibly inconsistent breakpoints across the framework.

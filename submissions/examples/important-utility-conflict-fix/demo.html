Utility Classes vs Component Styles — !important Conflict Fix

Previously, both utility classes (e.g. .u-hidden, .u-text-center) and component styles (e.g. .ease-card, .ease-alert) used !important on overlapping properties. Since two !important declarations of equal specificity resolve by source order, utility classes randomly failed to override components depending on stylesheet load order — defeating the entire purpose of a utility class ("always wins").

1. Utility overrides now win reliably

This card component sets its own display and text-align. Toggling .u-hidden / .u-text-center on it now correctly overrides the component's own styling, every time.

This card has component-level text-align: left and display: block baked in.

2. Component internal states no longer fight utilities

The alert below has its own internal "dismissed" state (also previously implemented with !important). It no longer collides with the .u-hidden utility — both now use ordinary specificity, with utilities living in a dedicated cascade layer that always wins.

This is a dismissible alert.

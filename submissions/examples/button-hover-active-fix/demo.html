ease-button — Hover vs Active State Conflict Fix

Two competing declarations previously fought over :hover and :active styling on the same button — both rules had equal specificity, so the cascade order (whichever appeared LAST in the stylesheet) decided the outcome rather than intent. This meant active (pressed) styling could be silently overridden by hover styling, or vice versa, depending on build order.

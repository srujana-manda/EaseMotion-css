ease-transition Shorthand vs Individual transition-property — Conflict Fix

Previously, the .ease-transition utility set the transition shorthand (e.g. transition: all 300ms ease;), while some components separately declared individual longhand properties like transition-property: transform; or transition-duration: 500ms; on the same element. Since shorthand properties reset ALL of their longhand sub-properties when applied, whichever rule came later in the cascade silently wiped out the other's intent — sometimes disabling a component's custom easing curve entirely, or reverting a utility's duration back to a component-only value.

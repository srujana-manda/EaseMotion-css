📊 ease-scroll-progress-bar

Reading Progress Bar

Scroll down to see the progress bar fill. It uses native CSS scroll-driven animations — no JavaScript scroll listeners, no requestAnimationFrame, no IntersectionObserver. Just a single div and a few CSS lines.

💡 Use the buttons above to preview different variants without reloading.

How It Works

animation-timeline: scroll(root block)

The bar uses animation-timeline: scroll(root block) to tie the progress animation directly to the document scroll position. The keyframe simply scales the bar from 0 to 1 on the X axis, creating the fill effect.

This is a modern CSS-first approach that completely eliminates the need for scroll event listeners. The browser handles the progress calculation natively, making it extremely performant even on long pages.

Available Variants

.ease-scroll-progress-bar--top (default)

Fixed to the very top of the viewport, sits above sticky navigation. The most common pattern used on blogs, documentation sites, and long-form articles.

.ease-scroll-progress-bar--bottom

Fixed to the bottom of the viewport. Useful when you have a top navigation bar you don't want to overlap, or for a different visual treatment.

.ease-scroll-progress-bar--pill

Adds border-radius and left/right margin to create a floating pill shape. Works great with gradient fills and glow effects for a modern look.

.ease-scroll-progress-bar--gradient

Replaces the solid color fill with a green-to-cyan gradient. Combine with --glow for an eye-catching neon effect. Multiple gradient presets are available.

.ease-scroll-progress-bar--glow

Adds a CSS box-shadow glow that matches the bar color. Works best with vivid colors and gradient variants on dark backgrounds.

.ease-scroll-progress-bar--track

Adds a subtle background rail using a ::before pseudo-element so readers can see the full extent of the progress bar even before they scroll.

Height Variants

Four height options: --thin (2px), --medium (4px, default), --thick (6px), and --xl (10px). Each is controlled by a CSS custom property so you can override freely.

Color Variants

Eight solid color variants: green (default), blue, purple, pink, orange, red, and white. Three gradient presets: green-cyan, purple-pink, and orange-red sunset. Plus a full rainbow gradient.

Theme Support

All variants automatically adapt under data-theme="neon", data-theme="dracula", and data-theme="dark". The bar color switches to match each theme's primary accent.

Accessibility

The bar element uses role="progressbar" with aria-label="Reading progress". Under prefers-reduced-motion, the animation is disabled and the bar shows at full width with reduced opacity — no scroll-linked motion.

Browser Support

animation-timeline: scroll() is supported in Chrome 115+, Edge 115+, and Firefox 110+ (with flag). Safari support is rolling out. In unsupported browsers the bar simply shows as a static full-width line — always a safe fallback.

Use Cases

Blog and article reading progress, documentation page navigation, long-form landing pages, onboarding flow step indicators, e-commerce product description pages, and any content-heavy page where users benefit from knowing how far they have scrolled.

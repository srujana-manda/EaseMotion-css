ease-speed-* Utilities vs Component Defaults — Duration Conflict Fix

Previously, components like ease-fade and ease-slide hardcoded their own animation-duration / transition-duration values, while utility classes like .ease-speed-fast and .ease-speed-slow used a SEPARATE set of duration numbers. Applying a speed utility to a component sometimes did nothing visible, because the component's own duration (often set with equal specificity) silently won depending on source order.

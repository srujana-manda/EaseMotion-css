How it works

The sticky header uses IntersectionObserver to detect when the element exits the viewport (scrolled past the top). At that point, the .ease-sticky-scrolled class is added, which triggers a CSS transition on box-shadow and optionally on background-color.

Why IntersectionObserver?

Unlike scroll event listeners, IntersectionObserver runs off the main thread and does not cause layout thrashing. It is the performant way to detect element visibility.

Transparent variant

Use .ease-sticky-header-transparent for headers that start transparent and gain a background on scroll — perfect for hero overlays.

Core Fix Proposal for Issue #20370

components/announce-bar.css assigns animation: ease-slide-down 0.3s ease but ease-slide-down is a utility class — the actual keyframe is ease-kf-slide-down.

Before (Broken)

animation: ease-slide-down 0.3s ease; /* No @keyframes ease-slide-down exists */
Your session will expire in 5 minutes

Missing keyframe — no slide-down animation

After (Fixed)

animation: ease-kf-slide-down 0.3s ease both; /* Matches @keyframes ease-kf-slide-down */
Your session will expire in 5 minutes

Keyframe exists — slides down correctly

prefers-reduced-motion Support

The fix also adds a @media (prefers-reduced-motion: reduce) guard so the entrance animation is disabled for accessibility.

@media (prefers-reduced-motion: reduce) { .ease-announce-bar { animation: none; } }

See README.md for the exact proposed changes to components/announce-bar.css.

Card Image Aspect Ratio

Fixing inconsistent card heights with aspect-ratio: 16/9 and object-fit: cover — Issue #14158

Broken — No aspect-ratio enforcement
Landscape (4:3)
Images with different natural aspect ratios stretch or shrink, causing uneven card heights in a grid.
Portrait (3:4)
Tall images push the card content down. The grid alignment breaks.
Square (1:1)
Each card has a different height because image dimensions are not constrained.
Fixed — With aspect-ratio: 16/9 + object-fit: cover
Landscape (4:3)
Forced to 16:9, content area fills remaining space. Cards align perfectly.
Portrait (3:4)
Cropped to fit 16:9. Card height matches its neighbors.
Square (1:1)
Uniform card grid maintained regardless of original image dimensions.
Configurable Ratio
1:1 Square
Override per image: style="aspect-ratio:1/1"
21:9 Ultrawide
style="aspect-ratio:21/9"
4:3 Classic
style="aspect-ratio:4/3"
/* Proposed fix for components/cards.css */ .ease-card-image { width: 100%; aspect-ratio: var(--ease-card-image-ratio, 16 / 9); object-fit: cover; display: block; } /* Configurable per card grid */ .card-grid { --ease-card-image-ratio: 4 / 3; }

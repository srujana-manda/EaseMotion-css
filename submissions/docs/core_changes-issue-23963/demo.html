ease-validity

CSS utility classes for :user-invalid / :user-valid pseudo-classes

Demo: Form Validation Styling

Try submitting the form or interacting with each field. The :user-invalid and :user-valid pseudo-classes style inputs after the user has interacted with them.

Please enter a valid email address.
Must be a valid URL (e.g. https://example.com).
Must be between 18 and 99.
This field is required.
Note: The :user-invalid and :user-valid pseudo-classes (vs :invalid / :valid) only apply after the user has interacted with the field — preventing premature error styling. These are supported in modern browsers (Chrome, Firefox, Safari 16+). For wider support, consider a JS polyfill.

Usage

<!-- Add both classes to any form control --> <input type="email" class="ease-validity-error ease-validity-success" required> /* The :user-invalid class gives a red border */ /* The :user-valid class gives a green border */

Interactive Docs

Introduction

This template demonstrates how to build interactive documentation using plain HTML, Tailwind CSS for utility-first styling, and EaseMotion CSS for smooth animations — zero build tools, zero framework overhead.

⚡

Lightweight by Default

No React, no Next.js — just static files that work anywhere.

🎨

Tailwind + EaseMotion

Utility classes for layout, animation classes for micro-interactions.

🔧

Extensible

Drop in a framework later if needed — the content is framework-agnostic.

Setup

Create a single HTML file, link Tailwind via CDN, and add EaseMotion CSS. That is the entire setup.

HTML 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <link href="https://cdn.jsdelivr.net/npm/easemotion-css@latest/easemotion.min.css" rel="stylesheet">
  <link href="https://cdn.jsdelivr.net/npm/tailwindcss@latest/dist/tailwind.min.css" rel="stylesheet">
</head>
<body class="p-8">
  <h1 class="ease-fade-in text-3xl font-bold">Hello, Docs!</h1>
</body>
</html>

Usage

Combine Tailwind utility classes for layout and spacing with EaseMotion animation classes for entry effects, hover states, and transitions.

📖 Tips for combining Tailwind + EaseMotion ▶

Use Tailwind for everything structural: grids, spacing, typography, colors. Use EaseMotion CSS classes (ease-fade-in, ease-slide-up, ease-scale) for animations. They work together without conflicts — Tailwind handles the canvas, EaseMotion paints the motion.

HTML 
<div class="ease-fade-in grid grid-cols-3 gap-4">
  <div class="ease-hover-lift bg-white rounded-xl p-4 shadow-sm">
    <h3 class="font-semibold">Card Title</h3>
    <p class="text-sm text-gray-600">Content here</p>
  </div>
</div>

Animations

EaseMotion provides drop-in animation classes that work alongside Tailwind. Here are some commonly used ones in docs:

CSS 
.ease-fade-in     { animation: easeFadeIn 0.5s ease forwards; }
.ease-slide-up    { animation: easeSlideUp 0.5s ease forwards; }
.ease-hover-lift  { transition: transform 0.2s ease, box-shadow 0.2s ease; }
.ease-hover-lift:hover { transform: translateY(-4px); }

Interactive Elements

Small vanilla JS snippets handle interactivity without frameworks. The copy button above is one example — here is the pattern:

JavaScript 
function copyCode(btn) {
  const code = btn.getAttribute("data-code");
  navigator.clipboard.writeText(code);
  btn.textContent = "✓ Copied!";
  btn.classList.add("copied");
  setTimeout(() => {
    btn.textContent = "📋 Copy";
    btn.classList.remove("copied");
  }, 2000);
}

Theming

Dark mode is handled by toggling a class on the body. CSS variables and Tailwind's dark: prefix handle the rest.

🌗 Theme toggle implementation ▶

A single class on <body> switches everything. CSS transitions on color and background properties make the switch smooth. The toggle button reads the current state and adjusts its label.

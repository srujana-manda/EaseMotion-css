CSS Quality CI Pipeline

Three automated checks run on every push and pull request to main, preventing broken CSS, enforcing variable consistency, and keeping the bundle lean.

Checks

1
CSS Lint Stylelint
Runs stylelint on all .css files using the project's .stylelintrc.json config. Catches syntax errors, duplicate properties, empty blocks, and invalid values.
npx stylelint "**/*.css" --config .stylelintrc.json --allow-empty-input
2
Bundle Size Budget
Ensures easemotion.min.css stays under 180 KB raw and 30 KB gzipped. Cross-platform stat for Linux and macOS.
SIZE=$(stat -c%s easemotion.min.css) # Linux SIZE=$(stat -f%z easemotion.min.css) # macOS GZ_SIZE=$(gzip -c easemotion.min.css | wc -c | tr -d ' ')
3
Custom Property Validation Reference
Scans all core/ and components/ CSS files and ensures every --ease-* custom property referenced is defined in core/variables.css.
DEFINED=$(grep -oP '--ease-[\w-]+(?=\s*:)' core/variables.css | sort -u) USED=$(find core components -name '*.css' ! -name 'variables.css' | \ xargs grep -roP '--ease-[\w-]+' | grep -oP '--ease-[\w-]+' | sort -u) # Fail if any USED variable is not in DEFINED

Workflow

name: CSS Quality Checks
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - name: Stylelint
        run: npx stylelint "**/*.css" --config .stylelintrc.json --allow-empty-input

  bundle-size:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check bundle size
        run: |
          FILE="easemotion.min.css"
          RAW_LIMIT=180000
          if [ -f "$FILE" ]; then
            SIZE=$(stat -c%s "$FILE")
            if [ "$SIZE" -gt "$RAW_LIMIT" ]; then
              echo "❌ $FILE is $SIZE bytes (limit: $RAW_LIMIT)"
              exit 1
            fi
          fi
      - name: Check gzipped size
        run: |
          FILE="easemotion.min.css"
          GZIP_LIMIT=30720
          if [ -f "$FILE" ]; then
            GZ_SIZE=$(gzip -c "$FILE" | wc -c | tr -d ' ')
            if [ "$GZ_SIZE" -gt "$GZIP_LIMIT" ]; then
              echo "❌ $FILE gzipped is $GZ_SIZE bytes (limit: $GZIP_LIMIT)"
              exit 1
            fi
          fi

  validate-vars:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check custom property references
        run: |
          DEFINED=$(grep -oP '--ease-[\w-]+(?=\s*:)' core/variables.css | sort -u)
          USED=$(find core components -name '*.css' ! -name 'variables.css' | \
            xargs grep -roP '--ease-[\w-]+' | grep -oP '--ease-[\w-]+' | sort -u)
          MISSING=()
          for var in $USED; do
            if ! echo "$DEFINED" | grep -qF "$var"; then
              MISSING+=("$var")
            fi
          done
          if [ ${#MISSING[@]} -gt 0 ]; then
            echo "❌ Undefined CSS variables:"
            printf '  %s\n' "${MISSING[@]}"
            exit 1
          fi

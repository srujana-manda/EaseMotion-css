prefers-reduced-motion Support

EaseMotion CSS now respects the user's motion preference via the prefers-reduced-motion media query. All animations, transitions, and hover effects are disabled when reduced motion is preferred, with an opt-in .ease-motion-safe class for essential animations.

CSS Variable — --ease-animation-iterations

The --ease-animation-iterations variable defaults to infinite normally but changes to 1 when prefers-reduced-motion: reduce is active. All looping animation classes (.ease-bounce, .ease-pulse, .ease-rotate, .ease-wave, .ease-ping) use var(--ease-animation-iterations, infinite) as their iteration count.

/* Looping animation uses the variable */
.ease-bounce {
  animation: ease-kf-bounce 1s var(--ease-animation-iterations, infinite);
}

/* When reduced motion is active, iterations drop to 1 */
@media (prefers-reduced-motion: reduce) {
  :root {
    --ease-animation-iterations: 1;
  }
}

Usage

Include the CSS after the main EaseMotion stylesheet. The reduced-motion rules activate automatically based on the user's OS/browser setting.

<!-- Include after easemotion.min.css -->
<link rel="stylesheet" href="style.css">

<!-- An element that always animates (opt-in) -->
<div class="ease-motion-safe ease-bounce">Essential</div>

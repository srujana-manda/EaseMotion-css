Expandable accordion panels with smooth animations — CSS-only, zero JavaScript.

Checkbox-based Accordion

Uses hidden <input type="checkbox"> with :checked ~ selector. Click a header to toggle its panel. Multiple panels can be open simultaneously.

EaseMotion CSS is a zero-dependency, animation-first CSS framework. It provides readable utility classes like ease-fade-in, ease-slide-up, and ease-hover-grow — no build step required.

The checkbox-based variant is keyboard accessible via <label> association. For screen reader support, consider the details/summary approach or add aria-* attributes with JavaScript.

Details/Summary Variant

Use .ease-accordion-simple with native <details>/<summary> elements for semantic HTML and built-in keyboard accessibility.

What is included in the framework?

The framework includes over 500 utility classes covering animations, transitions, layout, typography, spacing, colors, and more. All classes are prefixed with ease- to avoid conflicts.

How do I customize the theme?

Customize using CSS custom properties. Override --ease-color-primary, --ease-radius-md, --ease-spacing-*, and other tokens in your stylesheet to match your brand.

Does it support dark mode?

Yes. EaseMotion CSS includes built-in dark mode support. Add class="dark" on the <html> element and styles automatically adapt via --ease-color-* variable overrides.

Usage Notes

Checkbox variant: Place <input type="checkbox" class="ease-accordion-checkbox"> as the first child of .ease-accordion-item, followed by a <label class="ease-accordion-header"> and .ease-accordion-content. Multiple panels can be open at once. Wrap everything in .ease-accordion.

Details/Summary variant: Add .ease-accordion-simple to .ease-accordion. Use native <details> and <summary> elements. The content goes in .ease-accordion-content. Only one panel can be open at a time by default — add name attribute to <details> for named groups.

Animations: Content panels animate max-height and opacity for smooth expand/collapse. Arrow rotates 180 degrees. All animations respect prefers-reduced-motion: reduce.

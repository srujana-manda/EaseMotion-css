Slide Direction Auto-Swap

When the page direction changes, ease-slide-in-left and ease-slide-in-right automatically swap. Click the play button to see the effect, then switch the language and play again.

Utilities

Logical Alignment: ease-start & ease-end

These utilities use CSS logical values (text-align: start / end) which automatically respect the document direction.

With dir="ltr" (English)

text-align: start (ease-start) Aligned to the left in LTR context
text-align: end (ease-end) Aligned to the right in LTR context

With dir="rtl" (Arabic)

text-align: start (ease-start) محاذاة إلى اليمين في سياق RTL
text-align: end (ease-end) محاذاة إلى اليسار في سياق RTL
Overrides

Explicit Direction Override: ease-rtl & ease-ltr

Use .ease-rtl to force RTL behavior on a component inside an LTR page, or .ease-ltr to keep a widget LTR inside an RTL page.

LTR page with an RTL card (.ease-rtl)

هذا النص من اليمين إلى اليسار يقرأ هذا القسم من اليمين إلى اليسار بفضل الصنف .ease-rtl

RTL page with an LTR code block (.ease-ltr)

// This code stays LTR even in RTL context
const greeting = "Hello World";
Logical Properties

Margin / Padding / Border Utilities

These use CSS logical properties (margin-inline-start, padding-inline-end, border-inline-start) that automatically adapt to direction.

Reference

CSS Implementation

/* RTL mirroring — swap left ↔ right animations */
[dir="rtl"] .ease-slide-in-left {
  animation-name: ease-kf-slide-in-right;
}

[dir="rtl"] .ease-slide-in-right {
  animation-name: ease-kf-slide-in-left;
}

/* Utility classes */
.ease-rtl { direction: rtl; }
.ease-ltr { direction: ltr; }
.ease-start { text-align: start; }
.ease-end { text-align: end; }

Screen Reader Only (sr-only)

Accessibility utilities for hiding content visually while keeping it available to assistive technologies. Press Tab to reveal the skip link.

Overview

Why sr-only Matters

The .ease-sr-only utility hides content from sighted users while keeping it present in the accessibility tree. This is essential for WCAG 2.1 AA compliance and enables developers to provide descriptive text for visual-only elements.

Display Visual Screen Reader Method
display: none Hidden Hidden Completely removed
visibility: hidden Hidden Hidden Inaccessible to all
aria-hidden="true" Visible Hidden Decoration only
.ease-sr-only Hidden Read Accessible hidden
Use Case 1

Icon-Only Buttons

Icon buttons need accessible labels. Without visible text, screen reader users cannot determine the button's purpose. Use .ease-sr-only inside icon buttons to provide a text label that only assistive technology reads.

Without sr-only ❌

No visible label — inaccessible

With sr-only ✅

Label hidden visually, read by screen readers
⚠

Tip: Even with .ease-sr-only, always pair with aria-label on the button itself for maximum compatibility.

Use Case 2

Skip Navigation Links

Skip links let keyboard users bypass repetitive navigation. The link should be hidden until focused. .ease-sr-only-focusable does exactly this — hidden by default, visible on tab focus.

How it works

Press Tab right now. The "Skip to main content" link at the very top of this page should become visible. This is the .ease-sr-only-focusable pattern in action.

<!-- Hidden until focused -->
<a href="#main-content" class="ease-sr-only-focusable">
  Skip to main content
</a>
Use Case 3

Form Input Labels

When using placeholder-only inputs or icon-only form fields, always provide a visible label for screen readers using .ease-sr-only.

Search with accessible label

The label "Search" and button text "Submit search" use .ease-sr-only.

Use Case 4

Section Landmarks & Headings

Screen reader users navigate via headings and landmarks. If a section is visually clear but lacks a heading, add one with .ease-sr-only for assistive technology.

CSS Implementation

/* Visually hidden, accessible to screen readers */
.ease-sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border-width: 0;
}

/* Hidden until focused (skip links) */
.ease-sr-only-focusable:not(:focus):not(:focus-within) {
  /* same as .ease-sr-only */
}

.ease-sr-only-focusable:focus,
.ease-sr-only-focusable:focus-within {
  position: static;
  width: auto;
  height: auto;
  overflow: visible;
  clip: auto;
  white-space: normal;
}

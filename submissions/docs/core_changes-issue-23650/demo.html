Source Maps Generation

Proposed build infrastructure using clean-css to generate source maps for production debugging.

Proposed Architecture & Setup
1 Modularize Cascade Layers (core/layers.css)

Moving the cascade layers declaration to its own file allows all @import statements to sit at the absolute top of easemotion.css. This enables compiler compliance for standards-based minifiers.

/* core/layers.css */
@layer easemotion-base, easemotion-components, easemotion-utilities;
2 CleanCSS Compiling (scripts/build-minified-css.mjs)

Integrating clean-css handles local CSS import inlining, compression, and outputs standard-compliant source map files with relative paths. 

import CleanCSS from "clean-css";

const options = {
  inline: ["local"],
  sourceMap: true,
  rebase: false
};

// Outputs: easemotion.min.css.map alongside minified file
3 Production Mapping Comment

A reference mapping comment is dynamically appended to the end of easemotion.min.css to allow browser developer tools to fetch and load the original partial layouts.

/*# sourceMappingURL=easemotion.min.css.map */

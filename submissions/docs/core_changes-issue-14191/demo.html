Hover + Reduced Motion

Proposed fix for easemotion/hover.css — Issue #14191

Hover animations motion ON
scale(1.15)

box-shadow

skewX(-4deg)
gradient
With reduced motion motion OFF
When prefers-reduced-motion: reduce is active, all hover transitions, transforms, and box-shadows are disabled via:

@media (prefers-reduced-motion: reduce) {
  .ease-hover-grow,
  .ease-hover-lift { transition: none !important; transform: none !important; }
  .ease-hover-grow:hover,
  .ease-hover-lift:hover { transform: none !important; }
}

To test: Enable "Reduce motion" in your OS accessibility settings. Hover over the cards above — no movement or animation will occur.
Proposed core changes:
easemotion/hover.css — wrap all hover classes in @media (prefers-reduced-motion: reduce) with transition: none !important and transform: none !important on both the base and hover states to fully suppress motion for users who request it.

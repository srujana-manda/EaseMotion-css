Footer Keyframe Namespace Collision Demo

This demo reproduces the keyframe namespace collision described in Issue #27881. It demonstrates how generic keyframe names such as float and heartbeat can conflict with application CSS and how prefixing them avoids these collisions.

Application Animation

The application defines a generic @keyframes float animation.

Current Footer (Collision)

The footer also defines @keyframes float. Since CSS keyframes are global, the framework and application may unintentionally override each other's animations.

Proposed Fix

Prefixing footer keyframes (for example, ease-kf-footer-float) isolates the animation while preserving the same visual behavior.

Heartbeat Animation

The same namespace collision can occur with @keyframes heartbeat. Using ease-kf-footer-heartbeat prevents conflicts with application or third-party styles.

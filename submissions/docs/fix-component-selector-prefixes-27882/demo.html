Fix: Non-ease-* Component Selector Prefixes

This proposal addresses Issue #27882 by introducing ease-* prefixed aliases for public component selectors that currently use generic class names. The existing selectors should remain as deprecated aliases to preserve backward compatibility.

Proposed Fix 1 — Breadcrumb

Proposed alias: .bg-dark → .ease-bg-dark

Active Breadcrumb

Proposed Fix 2 — Skeleton

Proposed alias: .skeleton-card-header → .ease-skeleton-card-header

Why This Change?

  • Improves consistency across public component selectors.
  • Reduces the chance of CSS selector collisions.
  • Aligns all component APIs with the ease-* naming convention.
  • Maintains backward compatibility through deprecated aliases.

Proposed Selector Mapping

Current Selector Proposed Selector
.bg-dark .ease-bg-dark
.skeleton-card-header .ease-skeleton-card-header

Existing selectors should remain available as deprecated aliases until a future major release to ensure existing applications continue to work without breaking changes.

Core Fix Proposal for Issue #20374

scripts/validate-bundle.mjs only validates easemotion.min.css but the build script also writes docs/easemotion.min.css — so the docs bundle can go stale undetected.

Before (Broken)

const bundlePath = path.join(rootDir, "easemotion.min.css"); // Only checks root bundle
Read easemotion.min.css
↓
Run build (rewrites both)
↓
Compare easemotion.min.css
↓
❌ docs/easemotion.min.css unchecked

Docs bundle can go stale silently

After (Fixed)

const bundlePaths = [ "easemotion.min.css", "docs/easemotion.min.css" ]; // Both bundles validated
Read both bundles
↓
Run build (rewrites both)
↓
Compare both bundles
↓
✅ Both match — pass

All bundles validated & restored on failure

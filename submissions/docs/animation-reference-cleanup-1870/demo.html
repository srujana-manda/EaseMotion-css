Animation Documentation Cleanup Demo

This demo illustrates the documentation issue described in Issue #1870. It compares the current documentation, where animation information is repeated across multiple sections, with a proposed structure that consolidates animation details into a single reference.

Current Documentation

Animation classes and descriptions are repeated in multiple sections, increasing maintenance effort and making the documentation harder to navigate.

README Structure

README
│
├── Animations
│   ├── ease-fade-in
│   ├── ease-slide-up
│   ├── ease-zoom-in
│   └── ...
│
├── Animation Reference
│   ├── ease-fade-in
│   ├── ease-slide-up
│   ├── ease-zoom-in
│   └── ...

Proposed Documentation

Maintain a single animation reference while keeping the animations section focused on introducing the available animation categories.

README Structure

README
│
├── Animations
│   └── Overview
│
└── Animation Reference
    ├── ease-fade-in
    ├── ease-slide-up
    ├── ease-zoom-in
    └── ...

Benefits

  • Removes duplicated documentation.
  • Creates a single source of truth.
  • Improves readability and navigation.
  • Reduces maintenance effort.
  • Helps keep documentation consistent.

Expected Result

The proposed organization preserves all existing animation documentation while eliminating repeated content, making the README easier to maintain and navigate.
